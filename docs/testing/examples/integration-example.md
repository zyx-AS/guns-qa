# 登录接口与审计日志联动测试（IT-auth-log-001）

## 1. 测试目的
验证用户登录成功后，认证模块与审计日志模块之间的集成是否正常，确保登录结果与审计记录能够同步落库。

## 2. 测试方法与设计依据
本测试采用增量集成测试方法，重点验证认证服务、日志服务、数据库写入链路之间的数据流转是否正确。

## 3. 前置条件
- 参与模块：认证模块、审计日志模块、数据库访问层
- 集成方式：CI 自动集成验证
- 被测版本：待补充
- 测试环境：测试环境，MySQL 8，Java 17
- 测试数据：有效账号一组
- 依赖服务状态：数据库与日志服务均正常可用

## 4. 测试步骤
1. 使用有效账号调用登录接口。
2. 检查认证接口返回结果。
3. 查询审计日志表中是否新增对应记录。

## 5. 预期结果
1. 登录接口返回成功。
2. 认证结果中的用户标识正确。
3. 审计日志表成功写入一条与本次登录对应的记录。

## 6. 实际结果
本示例中，实际结果应重点说明接口返回是否正常、日志记录是否成功写入，以及两者的时间和用户标识是否一致。
