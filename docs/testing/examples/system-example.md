## 1. 测试目的
验证管理员从登录后台到新增用户成功保存的完整业务流程是否符合预期。

## 2. 测试方法与设计依据
本测试采用业务流程驱动的系统测试方法，覆盖用户登录、页面跳转、表单填写、保存提交和结果验证等关键步骤。

## 3. 前置条件
- 被测模块/页面：登录页、用户管理页、新增用户页
- 被测版本：待补充
- 测试环境：测试环境
- 浏览器/客户端：Chrome 最新版
- 测试账号与数据：管理员账号一组，新用户数据一组
- 依赖服务状态：认证、用户服务、数据库服务均正常

## 4. 测试步骤
1. 使用管理员账号登录系统。
2. 进入用户管理页面并点击“新增用户”。
3. 填写必填项后提交。
4. 检查页面提示和用户列表结果。

## 5. 预期结果
1. 系统允许管理员成功登录。
2. 新增用户页面正常打开。
3. 表单提交后给出成功提示。
4. 用户列表中出现新建用户数据。
