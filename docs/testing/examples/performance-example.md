## 1. 测试目的
验证登录接口在 50 并发用户场景下的响应时间、吞吐量、稳定性和错误率是否满足预期。

## 2. 测试方法与设计依据
本测试采用负载测试方法，以 50 并发用户持续访问登录接口，并与既有基线结果进行对比分析。

## 3. 前置条件
- 被测对象：登录接口 `/login`
- 被测版本：待补充
- 测试环境：测试环境
- 测试工具：JMeter
- 测试时长：10 分钟
- 并发用户数/线程数：50
- 测试数据规模：有效账号 200 组

## 4. 测试步骤
1. 准备测试环境与账号数据。
2. 配置 50 并发压测场景。
3. 执行压测并采集响应时间、吞吐量、错误率和资源使用情况。
4. 对比基线并输出分析结论。

## 5. 预期结果
- 平均响应时间：不高于 500 ms
- P95/P99：满足项目基线要求
- 吞吐量：满足项目基线要求
- 错误率：低于 1%
- CPU/内存/数据库连接占用：在可接受范围内
