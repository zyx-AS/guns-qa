# 测试 README 模板

所有测试 README 都建议先写一个元数据表，再按对应测试类型套用正文模板。

## 推荐的元数据表

```text
| 字段 | 内容 |
| --- | --- |
| 测试类型 | 单元测试 / 集成测试 / 系统测试 / 性能测试 |
| 测试编号 | `UT-xxx` / `IT-xxx` / `ST-xxx` / `PT-xxx` |
| 对应代码 | `guns-tests/src/test/...` |
| Jira | `GUNSQA-xxx` |
| GitHub 工作流 | `GUNS Unit Test` 或其他 workflow 名称 |
```

建议统一编号：
- 单元测试：`UT-模块-序号`
- 集成测试：`IT-链路-序号`
- 系统测试：`ST-场景-序号`
- 性能测试：`PT-场景-序号`

## 单元测试模板

```text

1. 测试目的
说明本测试要验证的类、方法或规则。

说明采用的测试方法或用例设计方法。
例如：等价类、边界值、判定表、异常分支覆盖、Mock 验证等。

3. 前置条件
- 被测类/方法：
- 被测版本：
- 测试环境：
- 测试工具：
- Mock/桩对象说明：

4. 测试步骤
1. ...
2. ...
3. ...

5. 预期结果
1. ...
2. ...
3. ...

6. 实际结果
说明本次执行的实际现象、返回值、断言结果、是否出现异常。

7. 结论
- 是否通过：
- 发现问题：
- 问题编号（Jira）：
- 是否影响后续测试：

8. 测试过程中的差异情况
- 差异内容：
- 原因：
- 对测试有效性的影响：
- 是否需要重测：
```

## 性能测试模板

```text
3.2 / 6.X.Y 测试名称（PT-xxx）

1. 测试目的
说明本测试要验证的性能目标。
例如：响应时间、吞吐量、稳定性、并发能力、资源占用。

2. 测试方法与设计依据
说明测试类型与设计方式。
例如：负载测试、压力测试、稳定性测试、并发测试、基线对比。

3. 前置条件
- 被测对象：
- 被测版本：
- 测试环境：
- 测试工具：
- 测试时长：
- 并发用户数/线程数：
- 测试数据规模：

4. 测试步骤
1. 准备环境与数据。
2. 配置压测场景。
3. 执行测试并采集指标。
4. 汇总与分析结果。

5. 预期结果
- 平均响应时间：
- P95/P99：
- 吞吐量：
- 错误率：
- CPU/内存/数据库连接占用：

6. 实际结果
按表格或文字说明实际测得指标，并与预期值或基线值对比。

7. 结论
- 是否达标：
- 性能瓶颈：
- 问题编号（Jira）：
- 是否影响系统发布：

8. 测试过程中的差异情况
- 差异内容：
- 原因：
- 对测试有效性的影响：
- 是否需要重新压测：
```
